# UI polish: MVP → полированный десктоп

Косметический апгрейд нового PySide6/QML-десктопа (`planner_desktop/`).
Изменён **только слой представления** (`qml/`). Бизнес-логика, персистентность,
синхронизация, контракты сервисов/ViewModel-ей и тесты не тронуты:
`187 passed` десктопных тестов, `307 passed` во всём наборе (единственный
провал — `test_settings_paths.py::test_macos_data_dir`, известный и не
связанный с UI: на Windows он падал и до этих правок).

Ни строки Python не изменено — все ViewModel-контракты
(`saveEditor`/`editorDataFor`/`toggleCompleted`/`deleteTask`/свойства-списки)
остались прежними, поэтому один `TaskEditorDialog` по-прежнему обслуживает и
«Сегодня», и «Календарь». Ни сети, ни бэкенда, ни автосинка не добавлено.

Скриншоты до/после — в [`screenshots/`](screenshots/).

## Дизайн-система

- **Токены** (`theme/Theme.qml`): добавлены тени/высота (для `MultiEffect`),
  единое семейство шрифта, градиенты акцента и фона, радиусы (вкл. pill),
  расширенная шкала отступов, цвета фокус-обводки и трека кольца. Компоненты
  и страницы по-прежнему не хардкодят hex.
- **Иконки** (`components/AppIcon.qml`): единый монохромный линейный набор,
  рисуется на `Canvas` (сетка 24×24, скруглённый штрих). Заменил разнородные
  цветные эмодзи в навигации, кнопках и карточках на согласованный стиль —
  без картинок-ассетов и без шрифтовых зависимостей.
- **Тени**: мягкая «высота» через `layer.effect: MultiEffect` на фоновом
  прямоугольнике `Panel`/карточек/диалогов — контент (текст) не растеризуется
  в слой и остаётся чётким.

| `Sidebar` | Настоящая навигационная панель: бренд-марка, линейные иконки, активный индикатор-полоска + мягкий pill, hover, футер-карточка. |

## Страницы

- **Сегодня**: `PageHeader`, ряд `StatTile`, компактный QuickAdd. На широком
  окне — двухколоночная раскладка: список задач слева, **правая сводка**
  справа (кольцо прогресса, карточка «Дальше» с ближайшей невыполненной
  задачей, «Ежедневные»). Раньше правая половина широкого окна пустовала —
  теперь занята. На узком окне сводка скрывается, контент центрируется, а
  «Ежедневные» показываются в основной колонке (ничего не теряется).
- **Календарь**: `PageHeader`, chevron-кнопки навигации, дни недели —
  приподнятые `Panel`-карточки с акцентом выбора и рамкой «сегодня»; список
  дня — карточки дизайн-системы.
- **Настройки**: `PageHeader`, карточки настроек с иконками, статус очереди
  синка с акцентной шапкой, информационный баннер с иконкой.
- **История**: `PageHeader` + пустое состояние (страница-заглушка, как и была).
- **Main**: единый шрифт приложения, мягкий градиент фона области контента,
  тост с иконкой, тенью и лёгким «подъёмом».

## Взаимодействие

hover/pressed/focus у кнопок, полей и карточек; плавные open/close диалогов;
анимация галочки и перехода выполнено; появление/скрытие тоста. Анимации
короткие и ненавязчивые (90–200 мс).

## Не входило в задачу

Реального Google-шлюза/OAuth/сети по-прежнему нет; формат хранения не менялся;
отложенные фичи (почасовая сетка + DnD, нативные пикеры, история, CRUD
ежедневных) остаются в [`FEATURE_PARITY.md`](FEATURE_PARITY.md).
