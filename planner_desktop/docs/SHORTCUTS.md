# Клавиатурные сокращения Planner Desktop

Справочник относится только к новому приложению `planner_desktop`.
Маршрутизация оконных shortcuts задаётся в `domain/keyboard.py`, применяется
через `UiStateViewModel` и не запускает Google-синхронизацию.

## Окно приложения

| Сочетание | Действие | Контекст |
|---|---|---|
| `Ctrl+N` | Новая задача | На Calendar подставляет выбранный день; с History/Settings переключает на Today |
| `Ctrl+Shift+N` | Новая задача со временем | Открывает редактор с безопасным ближайшим временным слотом; на Calendar использует выбранный день |
| `Ctrl+Alt+N` | Новая задача из шаблона | Открывает редактор и выбор локального шаблона; ничего не сохраняется до «Создать» |
| `Ctrl+K` / `Meta+K` | Фокус в Quick Add | Переключает на Today |
| `/` | Фокус в Quick Add | Только вне текстового поля и модального диалога |
| `Ctrl+R` | Обновить локальные модели | Перечитывает Today, Calendar, History, daily и Settings; **не** запускает Google sync |
| `Ctrl+F` / `Meta+F` | Открыть глобальный поиск / вернуть фокус в строку | Работает с любой страницы; повторное нажатие не сбрасывает запрос и фильтры |
| `Ctrl+D` | Дублировать одну выбранную задачу | Только вне текстового ввода; не срабатывает для tombstone; в Search требует ровно один выбранный результат |

## Глобальный поиск и множественный выбор

| Клавиша / жест | Действие | Контекст |
|---|---|---|
| `Up` / `Down` | Предыдущий / следующий результат | Поле поиска или список результатов |
| `Enter` | Открыть выбранный результат в общем редакторе | Search result list; отдельного CRUD-пути нет |
| `Ctrl+A` | Выбрать все видимые результаты | Только когда фокус в result list; не перехватывает Select All в тексте |
| `Ctrl+Click` | Переключить одну задачу в selection | Today, Calendar agenda/undated, Search и безопасные one-off rows History |
| `Shift+Click` | Выбрать непрерывный видимый диапазон | Anchor и порядок берутся из текущего отфильтрованного списка |
| `Space` | Переключить completion выбранной задачи | Только вне текстового ввода; daily history rows исключены |
| `Delete` | Открыть подтверждение bulk delete | При непустом multi-selection; удаление не выполняется без подтверждения |
| `Esc` | Закрыть вложенный диалог → очистить selection → очистить query → закрыть Search | Выполняется по одному детерминированному шагу |

## Выбранная задача и Calendar

| Клавиша | Действие | Контекст |
|---|---|---|
| `Enter` | Открыть редактор выбранной задачи | Today, Calendar или обычная задача в History, вне текстового поля |
| `Space` | Выполнить / вернуть / восстановить выбранную задачу | Today, Calendar или обычная задача в History, вне текстового поля |
| `Delete` / `Backspace` | Запросить удаление выбранной задачи | Обычные задачи на Today/Calendar/History; удаление всегда требует подтверждения |
| `Esc` | Отменить drag/resize; иначе очистить выбор | Если открыт popup/dialog, сначала закрывается он по своему правилу |
| `Left` / `Right` | Предыдущий / следующий день | Только когда открыта Calendar и фокус не в текстовом поле |
| `PageUp` / `PageDown` | Предыдущий / следующий период | В режиме «День» — соседний день; в рабочей/полной неделе — соседняя неделя |
| `Home` | Перейти к сегодня и один раз прокрутить к текущему часу | Только когда фокус находится в почасовой сетке Calendar |
| `Up` / `Down` | Предыдущее / следующее видимое событие | Только когда фокус находится в почасовой сетке; all-day идёт перед timed |
| `Alt+Up` / `Alt+Down` | Переместить selected timed task на 15 минут | Только Calendar; границы дня и минимальная длительность проверяет domain policy |
| `Alt+Shift+Left` / `Alt+Shift+Right` | Переместить selected task на один день | Для timed и all-day; span all-day сохраняется |
| `Alt+Shift+Up` / `Alt+Shift+Down` | Уменьшить / увеличить длительность на 15 минут | Только selected timed task; минимум 15 минут |
| `Ctrl+Alt+A` | Перевести selected scheduled task в all-day | Только Calendar |
| `Ctrl+Alt+U` | Перевести selected scheduled task в «Без даты» | Linked event ставит безопасную delete-операцию в локальную очередь |

## Редактор и пикеры

| Сочетание | Действие |
|---|---|
| `Ctrl+S` | Сохранить задачу; при ошибке валидации диалог остаётся открытым |
| `Esc` | Закрыть редактор или активный popup без сохранения |
| `Tab` до тумблера «Повторять», `Space` | Включить/выключить повторение новой задачи |
| Диалог «Область изменений серии» | `Tab` между «Только этот» / «Этот и все будущие» / «Отмена»; `Enter` подтверждает только кнопку с фокусом; `Esc` отменяет без изменений |
| Диалог «Связь серии с Google Calendar» | `Tab` достигает connect/actions/close; `Enter`/`Space` активирует только выбранное явное действие; `Esc` закрывает без изменения link |
| Диалог «Конфликт серии с Google Calendar» | Начальный фокус — на заголовке, поэтому `Enter` сразу после открытия ничего не подтверждает; `Tab` достигает трёх явных действий и «Закрыть»; каждое действие требует отдельного подтверждения; `Esc` отменяет без изменений |
| Диалог конфликта экземпляра Google | `Tab` достигает Use Google / Keep Planner / Keep both locally / Ignore; Keep Planner требует отдельного подтверждения; `Esc` оставляет quarantine без изменений |
| Подтверждение move/resize связанного экземпляра | commit выполняется только после кнопки «Изменить экземпляр»; `Esc` отменяет preview и сохраняет исходный `occurrence_key` |
| Диалог «Серия удалена в Google Calendar» | Те же правила: ничего не выбрано по умолчанию, `Enter` с начального фокуса не подтверждает, подтверждение обязательно, `Esc` отменяет |
| `Enter` | Подтвердить безопасное действие в поддерживаемом поле/контроле |
| `Tab` / `Shift+Tab` | Перейти между доступными контролами формы |
| `Enter`, `Space` или `Down` на дате | Открыть календарь выбора даты |
| `Left` / `Right`, `Up` / `Down` в календаре | Переместить выбор на день или неделю; `Enter` подтверждает, `Esc` закрывает popup |
| `Up` / `Down` на времени | Открыть список и выбрать соседний вариант; `Enter` подтверждает без сохранения всей формы |

## Контекстные правила безопасности

- Одиночные и Calendar interaction клавиши (`Enter`, `Space`, `Delete`,
  `Backspace`, стрелки, `/`, `Alt+…`, `Ctrl+Alt+…`)
  не перехватываются окном, пока пользователь печатает в `TextInput` или
  `TextEdit`.
- `Ctrl+N`, `Ctrl+Shift+N`, `Ctrl+K`, `Ctrl+F` и `Ctrl+R` не конфликтуют с обычным
  вводом и могут работать при фокусе в тексте, но все оконные shortcuts
  выключаются поверх открытого модального диалога.
- Редактор и popup-ы обрабатывают собственные `Ctrl+S`, `Enter` и `Esc`;
  окно не выполняет второе действие поверх них.
- Неизвестное сочетание безопасно игнорируется.
- Единственный пользовательский запуск Google-синхронизации в Phase 3.2B3B —
  явная кнопка «Синхронизировать сейчас» в Settings (или ручной CLI); ни
  `Ctrl+R`, ни открытие страницы, ни минутный таймер current-time line синк
  не запускают.
