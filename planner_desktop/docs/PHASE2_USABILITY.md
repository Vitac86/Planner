# Фаза 2: от «красивого MVP» к рабочему ежедневнику

> **Исторический снимок до текущего продуктового роадмапа.** Название
> «Фаза 2» здесь относится к старой итерации и не означает roadmap Phase 2.
> Актуальные статусы и следующие фазы: [`PRODUCT_ROADMAP.md`](PRODUCT_ROADMAP.md)
> и [`FEATURE_PARITY.md`](FEATURE_PARITY.md); текущие клавиши:
> [`SHORTCUTS.md`](SHORTCUTS.md).

Итерация юзабилити нового PySide6/QML-десктопа (`planner_desktop/`).
Цель — чтобы планировщик ощущался рабочим инструментом, а не просто
аккуратным списком. Архитектура (Domain → Repository → Service →
ViewModel → QML), локальность и тестовые гарантии сохранены.

**Жёсткие рамки соблюдены:** старое Flet-приложение не тронуто; OAuth/токены,
старый `sync_service.py`, боевой `app.db` — не открывались и не менялись;
сети/бэкенда/автосинка с Google не добавлено; формат хранения расширен
только аддитивно (две новые таблицы, `PRAGMA user_version` 2 → 3, всё через
идемпотентный `create_schema`).

## 1. Инспектор выбранной задачи (`components/TaskInspector.qml`)

- Одиночный клик по карточке выделяет задачу (видимая акцентная обводка),
  двойной — как раньше открывает редактор.
- На широком окне «Сегодня» правая колонка показывает инспектор:
  название, время/дата, приоритет, статус выполнения, статус синка
  (локальная / ждёт синка / связана с Google) и заметки.
- Действия из инспектора: выполнить/снять, изменить, удалить, закрыть.
- Если задача не выбрана — как и было: кольцо прогресса, «Дальше»,
  ежедневные. Выбор живёт в QML (`TodayPage.selectedUid`), Python не
  затронут; при удалении выбранной задачи выделение снимается.

## 2. Умный Quick Add (`domain/quick_parse.py`, `components/QuickAdd.qml`)

- Компактная строка осталась компактной; добавлен лёгкий чип приоритета
  (меню 0–3) — строка не «толстеет».
- Консервативный разбор ввода: ключевые слова `сегодня/завтра/послезавтра`
  и время `ЧЧ:ММ` (с предлогом «в/к» или без). Примеры:
  «Отчёт 15:00» → сегодня 15:00; «Позвонить Ивану завтра» → завтра, весь
  день; «Встреча сегодня 18:30» → сегодня 18:30. Всё нераспознанное
  остаётся в названии; если после вырезания токенов имя пустеет — задача
  создаётся без даты (не безымянной календарной).
- Enter создаёт задачу, Escape очищает поле / сворачивает «Детали».
- Валидация видимая и неблокирующая (ошибка под строкой, UI не виснет).
- Компактный ввод идёт через `TodayViewModel.addQuick(text, priority)`,
  развёрнутые «Детали» — через `addTaskDetailed(...)` с приоритетом. Старый
  7-аргументный `addTask` сохранён (обратная совместимость и тесты).

## 3. Настоящие ежедневные задачи (локально)

Заглушка-чек-лист заменена реальной сущностью:

- `domain/daily_task.py` — `DailyTask` (название, вкл/выкл, 7-битная маска
  дней недели, заметки, предпочтительное время, тумбстоун); `occurs_on(date)`.
- Хранилище: таблицы `desktop_daily_tasks` и `desktop_daily_completions`
  (отметка выполнения по дате). Репозитории in-memory и SQLite с одним
  контрактом; `usecases/daily_task_service.py` — валидация, вхождения на
  дату, отметки; `viewmodels/daily_tasks_viewmodel.py` + `DailyTasksDialog`.
- Ежедневные появляются на «Сегодня», когда подходит день недели, и
  отмечаются отдельно от разовых задач; отметка хранится по дате
  (переход через полночь не нужен — каждый день чистый).
- Полностью локально: в Google Calendar не уходят, Calendar-операций не
  порождают.
- Тесты: маска дней, появление на сегодня, отметка на конкретную дату,
  персистентность SQLite (переоткрытие БД), CRUD и ViewModel.

## 4. Юзабилити «Календаря» (`pages/CalendarPage.qml`)

- Недельная полоса сохранена.
- Сводка выбранного дня: бейдж «N из M готово» + быстрая кнопка «Задача».
- Вертикальная агенда дня: временная колонка, точка-маркер и линия-таймлайн
  слева, справа — те же `TaskCard`. Список сортируется «весь день» →
  по времени начала (сортировка в `CalendarViewModel._tasks_for`).
- Создание задачи с подставленной датой выбранного дня — как раньше.
- Полноценная почасовая сетка и drag-and-drop сознательно отложены
  (см. FEATURE_PARITY.md) — здесь только безопасная агенда.

## 5. Клавиатура и взаимодействие (`qml/Main.qml`)

- `Ctrl+N` — новая задача (на «Календаре» — с датой выбранного дня).
- `Ctrl+K` и `/` — фокус в Quick Add (переключает на «Сегодня»); `/` не
  срабатывает, когда фокус в текстовом поле.
- `Enter` — сохранить в диалогах/Quick Add; `Escape` — закрыть диалог /
  свернуть Quick Add.
- `Delete`/`Backspace` — удалить выбранную задачу, только с подтверждением
  и только когда фокус не в поле.
- Фокус-обводки у кнопок/полей и явное выделение карточки.

## 6. Пустые состояния и микрокопирайт (`components/EmptyState.qml`)

- `EmptyState` получил кнопку действия. «Сегодня» без задач предлагает
  «Создать первую задачу»; день календаря без задач — «Создать задачу на
  этот день»; ежедневные без пунктов — «Настроить».
- «Настройки»: подзаголовок прямо говорит про локальное хранение и ручной
  синк; статус очереди и баннер об отсутствии автосинка сохранены.
