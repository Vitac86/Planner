# Feature parity: старое Flet-приложение → новый PySide6/QML десктоп

Инвентаризация по файлам старого UI (`ui/pages/today.py`,
`ui/pages/calendar.py`, `ui/dialogs.py`, `ui/daily_tasks.py`). Старые файлы
использованы **только как справка по поведению** — они не изменялись.

Статусы: `implemented` — есть в новом приложении; `partial` — есть в
урезанном виде; `missing` — нет, планируется; `deferred` — сознательно
отложено (с причиной).

Фазы: **MVP** — этот коммит; **Phase 2** — следующая итерация UI;
**Phase 3+** — после подключения реального Google-шлюза.

> Визуальный апгрейд MVP-интерфейса (дизайн-система, тени, иконки, правая
> сводка «Сегодня», компактный Quick Add) описан в
> [`UI_POLISH.md`](UI_POLISH.md). Бизнес-логика и контракты не менялись.
>
> Юзабилити-итерация фазы 2 (инспектор задачи, умный Quick Add с
> приоритетом, реальные ежедневные задачи, агенда календаря, клавиатурные
> сокращения, живые пустые состояния) описана в
> [`PHASE2_USABILITY.md`](PHASE2_USABILITY.md).
>
> Итерация «История + фильтры календаря + диагностика» добавила: реальную
> страницу «История» (журнал выполненного из разовых задач по
> `Task.completed_at` и отметок ежедневных, фильтр 7/30/всё, «вернуть
> в работу»), фильтры агенды календаря (все / активные / выполненные /
> ежедневные) с чек-листом ежедневных на выбранный день и инспектором,
> разбивку очереди и панель «Диагностика» в настройках. Схема БД v3 → v4
> (аддитивная колонка `tasks.completed_at`, backfill из `updated_at`).

## Переходы состояния задачи

| Переход | Статус | Примечание |
|---|---|---|
| Создать без даты | implemented | локальная задача, операций в очереди нет |
| Создать с датой и временем | implemented | ставит `create` в локальную Calendar-очередь |
| Создать all-day | implemented | ставит `create` (форма date/date) |
| Правка текста/приоритета | implemented | у синхронизированной задачи ставит `update` |
| Перенос даты/времени | implemented | через редактор |
| Без даты → запланирована | implemented | ставит `create` |
| Запланирована → без даты (unschedule) | partial | поддержано для задач, чьё событие ещё не допушено (снимается pending `create`), и для привязанных одиночных событий (ставится `delete` события, задача отвязывается и остаётся локальной). Для **экземпляра повторяющегося события** — сознательно запрещено с сообщением «пока не поддерживается» (уроки dead-letter старого приложения). |
| Выполнено/не выполнено | implemented | локально; событие в календаре не трогается (фаза 1) |
| Удаление | implemented | тумбстоун + `delete` в очередь, если событие существовало |
