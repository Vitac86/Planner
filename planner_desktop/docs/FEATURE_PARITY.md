# Feature parity: старое Flet-приложение → новый PySide6/QML десктоп

Инвентаризация по файлам старого UI (`ui/pages/today.py`,
`ui/pages/calendar.py`, `ui/dialogs.py`, `ui/daily_tasks.py`). Старые файлы
использованы **только как справка по поведению** — они не изменялись.

Статусы: `implemented` — есть в новом приложении; `partial` — есть в
урезанном виде; `missing` — нет, планируется; `deferred` — сознательно
отложено (с причиной).

Колонка «Фаза» указывает фазу продуктового роадмапа
([`PRODUCT_ROADMAP.md`](PRODUCT_ROADMAP.md)), в которой фича реализована
или запланирована: **Фаза 1** — UX ежедневного использования;
**Фаза 2** — полноценный календарный UI; **Фаза 3** — паритет функций
задач; **Фаза 4** — продуктизация синхронизации; **Фаза 5** — миграция
и релиз. Уже реализованная до принятия роадмапа базовая функциональность
отнесена к **Фазе 1**; исторические названия ранних итераций в таблицах
не используются.

> Ранняя визуальная итерация (дизайн-система, тени, иконки, правая
> сводка «Сегодня», компактный Quick Add) описан в
> [`UI_POLISH.md`](UI_POLISH.md). Бизнес-логика и контракты не менялись.
>
> Историческая юзабилити-итерация (инспектор задачи, умный Quick Add с
> приоритетом, реальные ежедневные задачи, агенда календаря, клавиатурные
> сокращения, живые пустые состояния) описана в
> [`PHASE2_USABILITY.md`](PHASE2_USABILITY.md).
>
> Итерация «История + фильтры календаря + диагностика» добавила: реальную
> страницу «История» (журнал выполненного из разовых задач по
> `Task.completed_at` и отметок ежедневных, фильтр 7/30/всё, «вернуть
> в работу»), фильтры агенды календаря (все / активные / выполненные /
> ежедневные) с чек-листом ежедневных на выбранный день и инспектором,
> разбивку очереди и панель «Диагностика» в настройках. Схема БД v3 → v4
> (аддитивная колонка `tasks.completed_at`, backfill из `updated_at`).
